# Criterios De Automatizacion Team360

Team360 clasifica oportunidades de automatizacion segun volumen, impacto, claridad de reglas, sistemas involucrados, riesgo y dependencia humana.

## Paquete Estandar

Un caso puede entrar en paquete estandar cuando el proceso tiene reglas claras, bajo riesgo, volumen suficiente y herramientas comunes como formularios, email, Excel, CRM o reportes simples.

El paquete estandar no debe incluir escritura critica, bypass de seguridad ni integraciones profundas no relevadas.

## Automatizacion Operativa

Un caso puede ser automatizacion operativa cuando hay volumen medio o alto, impacto operativo claro, varios sistemas digitales y reglas parcialmente claras.

La automatizacion operativa suele requerir aprobacion humana cuando hay ERP, datos sensibles, cambios de estado, carga de documentos o decisiones con impacto economico.

## Consultoria Previa

Un caso requiere consultoria cuando el proceso esta desordenado, tiene muchas excepciones, depende de juicio experto, usa datos sensibles, requiere permisos inciertos o involucra escritura critica en ERP.

La consultoria previa busca mapear proceso, roles, datos, permisos, riesgos y frontera entre automatizacion total, asistida y manual.

## No Recomendable

Un caso no es recomendable como automatizacion directa si el volumen es bajo, el impacto es bajo, el proceso no esta definido, se pide evadir controles de seguridad, o depende de hardware, biometria, firma fuerte o anti-bot agresivo.
